FAQ

Compiling

Q: I can't get elmer to compile, I've tried specifying compilers and libraries by hand to the correct ones with no avail.
A: If the platform and compiler is not listed in the "known to compile" section, then there is a big chance that the code won't compile. You can of course ask for help, but we won't guarantee anything :)

Q: Is there any way to disable cross dependencies for the fortran files in the fem module?
A: Yes there is. Go to subdirectory fem/src and type make nofdeps. If you need to re-enable the dependencies type make fdeps.

Q: Why are so many files compiled even though I have only made a small modification to the source in the fem module?
A: The fortran dependencies are needed for compiling the fem package the first time (so that files are compiled in correct order). After this, the modifications are often so small that you would actually just need to compile only modified files (if there are no major structural changes), you can achieve this by removing dependencies, see previous answer.

Q: I'm getting weird compilation errors when compiling with multiple threads eg. make -j42.
A: There are many causes for weird errors, but possibly the most common one is related with NFS not being able to handle the traffic amount generated by massively parallel compilation. Try using a mount point that isn't a networked file system or reduce the number of processes.

Q:How do i compile my own solver/material/bc/whaterver routines?
A: elmerf90 -o File File.f90
